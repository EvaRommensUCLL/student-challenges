Git basics

Add, commit en push

  • Open vscode in de folder waar je challenges staan (bv. c:\introductieproject\challenges)

  • Ga naar de folder "git\basics"

  • Maak een nieuw bestand "solution.txt" en zet op de eerste lijn jouw naam en op de tweede lijn jouw studentnummer.

  • In Git bash voer je volgend commando uit:

$ git status

  • Zoals je kan zien detecteert git nu het bestand. Maar het is nog niet toegevoegd aan onze lokale git repository. Om dit te doen doe je:

$ git add solution.txt
$ git commit -m "Commit solution file"

  • Via "git add" zeg je tegen git om het bestand te tracken, "git commit" gaat het bestand effectief toevoegen aan de lokale repository. Via "-m" voeg je een boodschap toe aan de commit.

  • Ga nu via een browser naar je repository (plak de link die nog op je klembord zou moeten staan). Zoals je kan zien staat hier nog steeds geen code, maar instructies om je repository te initialiseren. Dit komt omdat we de wijzigen die je net hebt gedaan op je lokale repository moet "pushen" naar de remote versie:

$ git push --set-upstream origin master

  • De optie "--set-upstream" zorgt ervoor dat er naar de remote repository wordt gepushed. Als je dit éénmaal hebt gedaan kan je de volgende keren gewoon "git push" uitvoeren.

  • Wanneer je nu gaat refreshen in de browser zal je wel jouw solution file (en commit message) zien staan.

  • In git bash kan je het status commando nog eens uitvoeren. Normaal gezien zou de output als volgt moeten zijn:

Your branch is up to date with 'origin/master'.

nothing to commit, working tree clean
